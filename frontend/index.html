How the pipeline works

Four agentic stages, with a human gate before simulation or export.

    High-impact edits awaiting sign-off

    Policies with two or more versions show detected changes. Upload the same policy again (e.g. 2025 then 2026 NCCI manual) — we version it automatically.

    Monitored sources

    Select a policy to view its summary, version-to-version changes, and drafted edits.
    Filter:
    Simulation runs approved edits against synthetic claims (no PHI). Upload a PPRRVU*.txt file from CMS MPFS for Medicare pricing.

    Upload a document

    Policy PDFs/DOCX run the pipeline. PPRRVU*.txt (CMS MPFS) loads Medicare pricing for Impact. GPCI/HCPCS data files are not policy text.

    Generate a mock policy

    Creates a synthetic policy in two versions (v1 + v2) so change detection and rule drafting fire on demand.

    Note on synthetic data

    Uploaded files run through the same pipeline as the seed corpus. Generated policies and claims are clearly labeled MOCK; dollar figures are illustrative estimates from a mock fee schedule, not real pricing.

    Run the pipeline on new policy text

    Paste a bulletin or policy excerpt. The agents summarize it, diff it against any prior version of the same document, and draft edits for SME review.
